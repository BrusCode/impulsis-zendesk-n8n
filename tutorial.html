📋 Visão Geral

O que é este projeto?

Sistema completo de disparo ativo WhatsApp via Impulsis com roteamento automático de tickets via N8N e encerramento sincronizado no Zendesk (WebPosto).

Fluxo Operacional

  1. Disparo Ativo: Agente envia mensagem via Impulsis para cliente (sessão WhatsApp encerrada)
  2. Ticket Original: Recebe tag impulsis_ativo e campo Retorno Ativo marcado
  3. Cliente Responde: Novo ticket é criado automaticamente pelo Sunshine Conversations
  4. IA-Follow: Detecta Retorno Ativo e adiciona tag impulsis_novo_contato
  5. Workflow 1 (N8N): Busca agente que enviou a mensagem via comentário Impulsis e roteia ticket novo
  6. Macro Encerramento: Agente finaliza atendimento e dispara Workflow 2
  7. Workflow 2 (N8N): Fecha ticket original, limpa campos e remove tags

✅ Novidade v3.0 - Assignee via Comentário Impulsis

Problema Resolvido: Agora o sistema identifica corretamente o agente que enviou a mensagem ativa, mesmo que seja diferente do assignee original.

Como funciona: O Workflow 1 busca comentários privados no ticket original, filtra pelo padrão do Impulsis ("Nome do Template:", "Destinatario:") e extrai o author_id do comentário como agente responsável.

Fallback: Se o comentário não for encontrado, utiliza o assignee original do ticket pai.

⚙️ Instalação

⚠️ Pré-requisitos

  • Zendesk com acesso Admin
  • N8N self-hosted (com endpoint acessível pelo Zendesk)
  • Sunshine Conversations configurado (WhatsApp)
  • Impulsis configurado para disparos ativos
  • Bot IA-Follow configurado

Passo 1: Clone o Repositório

git clone https://github.com/BrusCode/impulsis-zendesk-n8n.git
cd impulsis-zendesk-n8n

Passo 2: Crie Campos Customizados no Zendesk

  1. Admin Center → Objects and Rules → Tickets → Fields
  2. Crie os 4 campos conforme tabela acima
  3. Anote os IDs gerados (se diferentes, ajuste nos workflows)

Passo 3: Configure Credenciais no N8N

  1. Acesse N8N → Credentials → Add Credential → HTTP Basic Auth
  2. Nome: Zendesk Basic Auth
  3. User: seu-email@dominio.com/token
  4. Password: SEU_TOKEN_API_ZENDESK
  5. Salve

🔄 Workflows N8N

Workflow 1 - Roteamento Seguro v3

Funcionalidade

Roteia o novo ticket criado pela resposta do cliente para o agente que enviou a mensagem ativa.

Fluxo

  1. Recebe webhook do Zendesk (novo ticket com tag impulsis_novo_contato)
  2. Busca ticket original com impulsis_ativo
  3. NOVO v3: Busca comentários do ticket original (sort_order=desc)
  4. NOVO v3: Filtra comentário privado do Impulsis (pattern: "Nome do Template:" + "Destinatario:")
  5. NOVO v3: Extrai author_id do comentário como agente responsável
  6. NOVO v3: Fallback: usa assignee original se comentário não encontrado
  7. Valida duplicidade (já existe outro ticket filho ativo?)
  8. Busca dados completos do agente
  9. Atualiza ticket novo: assignee, status=open, tags, campos customizados, nota privada
  10. Adiciona nota no ticket original vinculando filho

Importação

1. N8N → Workflows → Import from clipboard
2. Cole o conteúdo de workflows/workflow1-roteamento.json
3. Substitua ZENDESK_BASIC_AUTH pela sua credencial
4. Ajuste endpoint webhook se necessário
5. Salve e ative

Workflow 2 - Encerramento

Funcionalidade

Fecha o ticket original quando o agente finaliza o atendimento no ticket novo.

Fluxo

  1. Recebe webhook do Zendesk (ticket novo com tag impulsis_encerrar_origem)
  2. Busca ticket original pelo campo ID Ticket Pai
  3. Atualiza ticket original: status=solved, limpa Retorno Ativo, remove impulsis_ativo
  4. Adiciona nota privada de encerramento

Importação

1. N8N → Workflows → Import from clipboard
2. Cole o conteúdo de workflows/workflow2-encerramento.json
3. Substitua ZENDESK_BASIC_AUTH pela sua credencial
4. Ajuste endpoint webhook se necessário
5. Salve e ative

🔧 Configuração Zendesk

Trigger 1 - Roteamento (Tag impulsis_novo_contato)

Condições

  • Ticket: Is | Created
  • Tags: Contains at least one | impulsis_novo_contato
  • Tags: Contains none | roteado_agente_automatico

Ações

  • Notify Webhook: https://seu-n8n.com/webhook/impulsis-retorno-routing-v3
  • Método: POST
  • Content Type: JSON
  • Body: 
    {
  "ticket_id": {{ticket.id}},
  "requester_id": {{ticket.requester.id}},
  "status": "{{ticket.status}}",
  "tags": {{ticket.tags}}
}

Trigger 2 - Encerramento (Tag impulsis_encerrar_origem)

Condições

  • Ticket: Is | Updated
  • Tags: Contains at least one | impulsis_encerrar_origem

Ações

  • Notify Webhook: https://seu-n8n.com/webhook/impulsis-encerramento
  • Método: POST
  • Content Type: JSON
  • Body: 
    {
  "new_ticket_id": {{ticket.id}},
  "origin_ticket_id": "{{ticket.ticket_field_41306351713940}}"
}

Configuração IA-Follow (Welcome Reply)

Validação de Retorno Ativo:

  1. No construtor de diálogo do IA-Follow, adicione validação no Welcome Reply
  2. Condicional: Se ticket tem campo Retorno Ativo (50639967936148) = true
  3. Ação: Adicionar tag impulsis_novo_contato
  4. Ação: NÃO enviar mensagem automática (pular Welcome)

Macro de Encerramento

Nome:

Encerrar Sessão Impulsis

Ações:

  • Status: Solved
  • Tags: Add impulsis_encerrar_origem
  • Nota privada: "Encerrando sessão Impulsis. Ticket original será fechado automaticamente."

  3. Validar: Comentário privado do Impulsis criado com dados do template

  9. Validar: Tags impulsis_retorno e roteado_agente_automatico adicionadas
  10. Agente: Finaliza atendimento e aplica macro "Encerrar Sessão Impulsis"
  11. Validar: Workflow 2 executa
  12. Validar: Ticket original fechado, campo Retorno Ativo limpo, tag impulsis_ativo removida

Troubleshooting

Ticket novo não roteado automaticamente

  • Verificar se tag impulsis_novo_contato foi adicionada pelo IA-Follow
  • Verificar trigger no Zendesk (condições e webhook)
  • Verificar logs de execução do Workflow 1 no N8N
  • Verificar se comentário privado do Impulsis existe no ticket original

Agente errado atribuído

  • Verificar se comentário do Impulsis contém "Nome do Template:" e "Destinatario:"
  • Verificar logs do nó "Extrair Agente Disparador do Impulsis" no Workflow 1
  • Se fallback está sendo usado, significa que comentário não foi encontrado

Ticket original não fecha

  • Verificar se macro adicionou tag impulsis_encerrar_origem
  • Verificar se campo ID Ticket Pai está preenchido no ticket novo
  • Verificar logs de execução do Workflow 2 no N8N
